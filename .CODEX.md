# CODEX Learning Guide

이 저장소는 논문 구현을 대화형으로 학습하기 위한 코드베이스다. 앞으로 이 프로젝트를 다룰 때의 목표는 단순 실행보다 **코드 흐름, 방법론, 실험 설계, 결과 해석을 단계적으로 마스터하는 것**이다.

## 학습 목표

- GPLR 알고리즘이 어떤 문제를 풀고, 어떤 가정 위에서 동작하는지 설명할 수 있게 한다.
- `persona_collection.ipynb`에서 LLM으로 persona label을 만들고 후처리하는 흐름을 이해한다.
- `gplr.ipynb`에서 prototype user sampling, RevAff affinity 계산, unlabeled user persona 생성까지의 전체 파이프라인을 추적한다.
- `src/random_walker/PageRank.py`와 `src/random_walker/ReverseRW.py`의 수식적 의미와 구현 선택을 연결해서 이해한다.
- LGCN3 / A_LGCN3 baseline이 GPLR 결과와 어떻게 비교되는지 파악한다.
- 논문 재현 또는 변형 실험을 할 때 어떤 파일, 데이터, 하이퍼파라미터를 바꿔야 하는지 판단한다.

## 프로젝트 지도

- `README.md`: 실행 방법과 주요 구성의 최상위 안내.
- `prompts.md`: persona 생성을 위한 LLM prompt 사례.
- `data/`: MBA 데이터, user-item-persona graph JSON, GPLR 결과 저장 위치.
- `src/persona_collection.ipynb`: LLM 기반 user persona 및 item-persona label 수집 파이프라인.
- `src/gplr.ipynb`: GPLR의 주 실행 노트북. 데이터 로드, prototype sampling, RevAff, 예측 결과 저장을 담당.
- `src/datasets/MBA.py`: MBA CSV를 읽고 user-item bipartite graph를 만든다.
- `src/random_walker/PageRank.py`: transition matrix, PageRank-style propagation, iterative sampler, persona prediction 구현.
- `src/random_walker/ReverseRW.py`: layered graph 구성과 reverse random walk 기반 RevAff 계산 구현.
- `src/random_walker/Evaluation.py`: 예측 결과 평가 유틸.
- `src/utils/GraphMaker.py`: 예측 persona를 graph 형식으로 다시 조립하는 유틸.
- `src/llm_connector/Collector.py`: OpenAI API 호출, prompt 구성, 응답 파싱 유틸.
- `src/baselines/lgcn3/`: LGCN3와 A_LGCN3 baseline 학습 코드.

## 핵심 코드 흐름

1. MBA 원본 데이터 로드
   - 시작점: `src/datasets/MBA.py::MBA_load_data`
   - 출력: `mba_df`, `user_ids`, `item_names`, `G_user`, `G_item`
   - 핵심 구조:
     - `G_user`: `{uidx: set(tidx)}`
     - `G_item`: `{tidx: set(uidx)}`

2. Persona label 수집
   - 시작점: `src/persona_collection.ipynb`
   - LLM이 user 구매 내역과 item 이름을 기반으로 persona를 고른다.
   - 주요 산출물:
     - `data/persona2idx.json`
     - `data/tri_graph_uidx2pidx.json`
     - `data/tri_graph_uidx2tidx_train.json`
     - `data/tri_graph_tidx2pidx.json`

3. Prototype user sampling
   - 시작점: `src/random_walker/PageRank.py::Iterative_Sampler`
   - RevAff 버전: `src/random_walker/ReverseRW.py::Iterative_Sampler_rev`
   - 목적: 모든 user를 LLM으로 라벨링하지 않고, 대표 prototype user만 선택한다.
   - 중요한 기준:
     - `criteria_rank_1`: 새 item coverage 중심.
     - `criteria_rank_3`: 현재 persona distribution과 후보 user의 예상 persona distribution을 이용.

4. Affinity propagation
   - PageRank 방식:
     - `graph2transP`가 user-user transition matrix `R`을 만든다.
     - `persona_prediction`이 labeled user에서 unlabeled user로 label score를 전파한다.
   - RevAff 방식:
     - `construct_graph`가 persona-user-item-user layered graph를 만든다.
     - `reverse_rw_3.reverse_random_walk_all`이 persona node에서 역방향 전파를 수행한다.
     - `persona_prediction_rev`가 unlabeled user별 persona affinity matrix를 만든다.

5. Application graph 생성
   - 시작점: `src/utils/GraphMaker.py::make_app`
   - 예측 score 상위 persona를 user별 label로 바꾸고 `uidx -> pidx` graph를 만든다.

6. Baseline 학습
   - 시작점: `src/baselines/lgcn3/run_exp.py`
   - README 기준:
     - graph filter 생성: `python3 pretraining/_graph_embeddings_tri.py`
     - LGCN3: `python3 run_exp.py --model 11 ...`
     - A_LGCN3: `python3 run_exp.py --model 14 ... --afd_alpha ...`

## 방법론을 이해할 때의 순서

1. 문제 정의부터 고정한다.
   - 입력은 user-item interaction graph다.
   - 비싼 LLM labeling을 모든 user에게 적용하지 않고 일부 prototype user에만 적용한다.
   - 목표는 나머지 user의 persona를 graph propagation으로 예측하는 것이다.

2. 데이터 구조를 먼저 따라간다.
   - `CustomerID -> uidx`
   - `Itemname -> tidx`
   - `persona string -> pidx`
   - 이후 대부분의 코드는 원래 ID가 아니라 index 기반 graph를 사용한다.

3. sampler를 분리해서 본다.
   - sampler는 "누구에게 LLM/persona oracle을 물어볼 것인가"를 정한다.
   - prediction은 "물어본 결과를 나머지 user에게 어떻게 퍼뜨릴 것인가"를 정한다.

4. PageRank와 RevAff를 비교한다.
   - PageRank 구현은 dense transition matrix `R` 중심이라 직관적이다.
   - RevAff 구현은 persona별 reverse walk를 사용해 필요한 affinity를 근사적으로 계산한다.
   - `e`는 reverse random walk의 approximation threshold로 읽는다.

5. baseline은 마지막에 본다.
   - GPLR의 데이터 산출물이 baseline 입력으로 어떻게 들어가는지 확인한 뒤 LGCN3 코드를 읽는다.

## 대화 학습 방식

앞으로 질문을 받으면 다음 방식으로 답한다.

- 먼저 관련 파일과 함수 위치를 짚는다.
- 그다음 데이터가 어떤 shape와 key 구조로 이동하는지 설명한다.
- 필요한 경우 수식적 의미를 코드 줄과 연결한다.
- 실행보다 이해가 목적이면, 바로 코드를 고치지 말고 흐름도와 작은 예시를 우선한다.
- 실험 재현이 목적이면, 실행 명령, 입력 파일, 출력 파일, 바뀌는 하이퍼파라미터를 명확히 분리한다.
- 코드에 버그나 재현성 위험이 보이면 학습 맥락에서 따로 표시한다.

## 주의할 점

- `persona_collection.ipynb`는 OpenAI API를 호출하므로 실행 비용이 발생할 수 있다.
- 일부 파싱 함수는 `eval`을 사용하므로 외부 입력을 그대로 실행하면 위험하다.
- notebook과 Python 모듈 사이에 중복 구현이 있을 수 있으니, 먼저 notebook 흐름을 본 뒤 모듈 함수를 확인한다.
- 실험 결과 재현에는 random seed, train/test split JSON, persona label JSON이 중요하다.
- baseline 쪽은 TensorFlow v1 compatibility mode를 사용한다.
